# ConexaoBiblioteca

A simple product management CRUD built with Node.js, Express, and Microsoft SQL Server. The app serves a static frontend from `public/` and exposes a REST API for managing products and viewing audit logs.

## Features

- Create, read, update, and delete products
- SQL Server persistence using the `mssql` driver
- Audit log generated by database triggers
- Static frontend served by Express
- CORS enabled for local development

## Project Structure

- `server.js` - Express server and API routes
- `public/index.html` - Frontend interface
- `schema.sql` - Database schema, triggers, and sample data
- `package.json` - Project scripts and dependencies

## Prerequisites

- Node.js 18+ recommended
- Microsoft SQL Server
- npm

## Setup

1. Install dependencies:

```bash
npm install
```

2. Create the database and tables:

Run `schema.sql` in SQL Server Management Studio, Azure Data Studio, or another SQL client connected to your SQL Server instance.

3. Update the database connection in `server.js`:

The current configuration is defined in `dbConfig` inside `server.js`. Replace it with your own SQL Server host, database, user, and password before starting the app.

## Running the Application

Start the server in production mode:

```bash
npm start
```

Or run it with automatic restart during development:

```bash
npm run dev
```

By default, the server runs at:

```text
http://localhost:3000
```

## API Endpoints

### Products

- `GET /api/produtos` - List all products
- `GET /api/produtos/:id` - Get a product by ID
- `POST /api/produtos` - Create a new product
- `PUT /api/produtos/:id` - Update an existing product
- `DELETE /api/produtos/:id` - Delete a product

### Logs

- `GET /api/logs` - List the most recent audit log entries

## Request Body Example

Use this payload for `POST /api/produtos` and `PUT /api/produtos/:id`:

```json
{
  "nome": "Mechanical Keyboard",
  "categoria": "Peripherals",
  "preco": 349.9,
  "estoque": 15
}
```

## Database Notes

The `schema.sql` file creates:

- `Produtos` table for product data
- `Produtos_Log` table for audit history
- Triggers for `INSERT`, `UPDATE`, and `DELETE`

The schema also inserts sample records so you can test the app right away.

## Security Note

The database credentials are currently hardcoded in `server.js`. For real deployments, move them to environment variables before publishing or sharing the project.
